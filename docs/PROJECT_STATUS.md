## 2. Стек и архитектура

- **Фронтенд:** чистый HTML5 / CSS3 / Vanilla JS, никаких фреймворков
- **Бэкенд:** `serve.py` — Python `HTTPServer` на порту 8080; отдаёт статику + эндпоинт `/api/notify`
- **Уведомления:** Telegram Bot API (через serve.py proxy) + SMTP email (через serve.py)
- **PDF спецификации:** генерация через `window.print()` (HTML → PDF через браузер)
- **Данные:** `data/catalog.json` (172 КБ), полностью статический
- **Хостинг сейчас:** VPS `202.148.53.107`, порт 8080, процесс запущен через `nohup`

## 4. Инфраструктура уведомлений

### Telegram (основной канал — работает)
- **serve.py** `/api/notify`: читает `proxy/.env`, форматирует сообщение, отправляет через Bot API
- Токен, chat_id — в `proxy/.env` (в .gitignore, не в репозитории)
- Rate limiting: не чаще 1 раза в 30 сек

### Email (работает частично)
- **serve.py** генерирует PDF через reportlab и отправляет через SMTP
- Credentials в `proxy/.env`: `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASS`, `NOTIFY_EMAIL`
- Функция есть, но требует проверки реальной SMTP-конфигурации

### Калькулятор — Email/Telegram (DEMO-режим)
- `calculator.js`: константы `EMAILJS_SERVICE_ID/TEMPLATE_ID/PUBLIC_KEY = 'DEMO'`
- `calculator.js`: `TELEGRAM_BOT_TOKEN/CHAT_ID = 'DEMO'`
- В DEMO-режиме показывает toast, реальной отправки нет
- **Требует настройки EmailJS аккаунта для email-отправки спецификаций**

## 7. Что НЕ сделано / требует внимания перед релизом

### Критично для релиза
1. **EmailJS для калькулятора** — зарегистрировать аккаунт на emailjs.com, создать шаблон, вставить реальные ключи в `calculator.js` (3 константы: SERVICE_ID, TEMPLATE_ID, PUBLIC_KEY)
2. **serve.py — однопоточный** — при одном зависшем запросе (SMTP timeout) весь сервер встаёт. Нужно переключить на `ThreadingHTTPServer`. Уже один раз упал (2026-04-08)
3. **Процесс serve.py не персистентен** — сейчас запущен через `nohup`, упадёт при перезагрузке VPS. Нужен `systemd` unit или supervisor
4. **privacy.html не в sitemap.xml** — страница есть, в карте сайта нет

### Желательно
5. **Реальные фото товаров** — сейчас везде SVG-заглушка «Фото отсутствует»
6. **product_backup.html** — артефакт, нужно удалить из корня
7. **Проверить SMTP** — убедиться что email-уведомления из форм заявок реально приходят
8. **Цены в catalog.json** — часть позиций `"price": null` («Цена по запросу»), уточнить с клиентом актуальность
